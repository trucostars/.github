# Contribuyendo a trucostars

¡Gracias por tu interés en contribuir a los proyectos de trucostars! 🎉

## Código de Conducta

Este proyecto y todos sus participantes están regidos por nuestro [Código de Conducta](CODE_OF_CONDUCT.md). Al participar, se espera que respetes este código.

## ¿Cómo Contribuir?

### Reportar Bugs

Si encuentras un bug, por favor abre un issue usando la plantilla de **Bug Report** y asegúrate de incluir:

- Una descripción clara del problema
- Pasos para reproducirlo
- El comportamiento esperado vs el actual
- Capturas de pantalla si aplica
- Información de tu entorno (OS, versión, etc.)

### Sugerir Mejoras

Las sugerencias de nuevas funcionalidades son bienvenidas. Abre un issue usando la plantilla de **Feature Request** con:

- Una descripción clara de la funcionalidad
- La motivación y el caso de uso
- Alternativas que hayas considerado

### Contribuir Código

1. **Fork** el repositorio
2. Crea una rama desde `main`:
   ```bash
   git checkout -b feature/mi-nueva-funcionalidad
   ```
3. Realiza tus cambios siguiendo las guías de estilo del proyecto
4. Asegúrate de que las pruebas pasen:
   ```bash
   # Ejecuta las pruebas del proyecto
   ```
5. Haz commit de tus cambios con mensajes descriptivos:
   ```bash
   git commit -m "feat: agregar nueva funcionalidad X"
   ```
6. Haz push a tu rama:
   ```bash
   git push origin feature/mi-nueva-funcionalidad
   ```
7. Abre un **Pull Request** usando nuestra plantilla

### Convención de Commits

Seguimos [Conventional Commits](https://www.conventionalcommits.org/):

| Prefijo    | Descripción                                      |
|------------|--------------------------------------------------|
| `feat:`    | Nueva funcionalidad                              |
| `fix:`     | Corrección de bug                                |
| `docs:`    | Cambios en documentación                         |
| `style:`   | Cambios de formato (sin afectar lógica)          |
| `refactor:`| Refactorización de código                        |
| `test:`    | Agregar o corregir pruebas                       |
| `chore:`   | Tareas de mantenimiento                          |

## Proceso de Revisión

- Un mantenedor revisará tu PR dentro de los próximos días hábiles
- Puede que se soliciten cambios o ajustes
- Una vez aprobado, el PR será mergeado a `main`

## Preguntas

Si tienes preguntas, puedes abrir una discusión en el [foro de la organización](https://github.com/orgs/trucostars/discussions).

¡Gracias por contribuir! 🚀
