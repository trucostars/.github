## Reportar una Vulnerabilidad

En trucostars tomamos la seguridad de nuestros proyectos muy en serio. Si descubres una vulnerabilidad de seguridad, te pedimos que nos la reportes de manera responsable.

**Por favor, NO abras un issue público para reportar vulnerabilidades de seguridad.**

### Cómo Reportar

Envía un correo a **security@trucostars.com** con la siguiente información (o utiliza la [herramienta de reporte privado de vulnerabilidades](https://github.com/trucostars/.github/security/advisories/new) de GitHub):

- Descripción del tipo de problema (ej. buffer overflow, SQL injection, cross-site scripting, etc.)
- Rutas completas del archivo(s) fuente relacionado(s) con la manifestación del problema
- Ubicación del código afectado (tag/branch/commit o URL directa)
- Cualquier configuración especial requerida para reproducir el problema
- Instrucciones paso a paso para reproducir el problema
- Prueba de concepto o código de exploit (si es posible)
- Impacto del problema, incluyendo cómo un atacante podría explotar el problema

### Tiempo de Respuesta

Nos comprometemos a:

- Confirmar la recepción de tu reporte dentro de **48 horas**
- Proporcionar una evaluación inicial dentro de **7 días**
- Mantenerte informado del progreso hacia una solución
- Notificarte cuando el problema esté resuelto

### Divulgación Responsable

Una vez que hayamos confirmado el problema y desarrollado una solución, trabajaremos contigo para coordinar la divulgación pública. Apreciamos tu paciencia y colaboración en este proceso.

Reconoceremos públicamente tu contribución (con tu permiso) en el aviso de seguridad asociado al problema.
